### Body
```
Anthropic launched Claude Skills in October 2025. Most non-technical people 
still don't understand what they are. Sharing how I figured it out and built 
21 marketing skills for Russian small businesses.

A Skill is a folder with one SKILL.md file. Frontmatter has 'name' and 
'description'. The body is plain markdown instructions for Claude.

You drop the folder in ~/.claude/skills/. Claude reads all skills at session 
start, picks the right one based on description matching, applies it.

How it differs from ChatGPT custom GPTs:
- Lives in a folder you control, not on OpenAI servers
- Plain markdown, you can git version it
- Sharable as a GitHub repo
- No subscription, no platform lock-in

Minimal SKILL.md:

---
name: my-first-skill
description: Writes a Telegram post for an expert consultant. 
Apply when user asks for a Telegram post.
---

# Instruction

When user asks for a Telegram post:
1. Ask for topic
2. Write hook in first line (question or specific fact)
3. 3-5 short paragraphs
4. CTA in last line

## Rules
- Paragraph 1-2 sentences
- Use "ты" not "вы"
- No emojis at start

That's it. Drop in folder, Claude finds it, applies it.

I packaged 21 such skills for marketing into open-source repo with MIT license. 
Schwartz/Ogilvy/Hopkins methodology baked into each one. 

Install:
git clone https://github.com/nibrovkina-cyber/natalia-marketing-department.git
mkdir -p ~/.claude/skills
cp -r skills/* ~/.claude/skills/

Then in Claude Code: /natalia-positioning

Happy to answer questions on writing your own skills.
```

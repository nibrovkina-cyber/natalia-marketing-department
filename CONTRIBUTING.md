# Contributing to natalia-marketing-department

Спасибо за интерес к проекту! Этот open-source repo — попытка дать малому бизнесу в РФ доступ к маркетинговому отделу без бюджета 100-200К ₽/мес. Чем больше людей форкает и улучшает — тем лучше для всех.

## Что особенно нужно

### 🟢 Высокий приоритет

1. **Реальные кейсы (showcase)** — твой бизнес или клиента, с цифрами. Формат:
   - Ниша + город
   - Что было ДО (метрика)
   - Какие skills использовал
   - Что стало ПОСЛЕ (метрика)
   - Сколько времени заняло
   
   Открой issue с тегом `showcase` или PR в `docs/showcase/`.

2. **Адаптации под другие рынки** — Беларусь, Казахстан, Грузия, Узбекистан. Локализация платежей, юр-требований, каналов трафика. Форкай → адаптируй → линкуй обратно.

3. **EN-версия skills** — для глобальной аудитории. Структура та же, контекст adapted под US/UK/EU. Существующая EN README в `docs/README.en.md`.

### 🟡 Средний приоритет

4. **Новые business types в `BUSINESS_TYPES`** — у тебя есть ниша которой нет в стандартном списке. Добавь и пришли PR.

5. **Адаптации под Claude Desktop** — большинство skills тестировались на Claude Code. Если ты пользуешься Claude Desktop и нашёл несовместимость — issue.

6. **MCP-обёртки для агентов** — Model Context Protocol позволяет интегрировать skills в другие IDE и среды. Желателен MCP-сервер для каждого агента.

### 🔵 Низкий приоритет (но полезно)

7. **Документация конкретных агентов** — расширенный README в каждой папке `skills/natalia-*/README.md`.

8. **Тесты для агентов** — unit-style проверки SKILL.md на корректность frontmatter, обязательные секции.

9. **CI / GitHub Actions** — автоматическая проверка SKILL.md при PR.

## Что НЕ нужно

❌ **Косметические правки русского текста без необходимости** — мы сознательно используем простой язык. Не переписывай "тебя" на "Вас".

❌ **Refactoring ради refactoring** — если skill работает, не трогай его без жалоб от пользователей.

❌ **Зависимости от платных SaaS** — repo должен работать локально с Claude Code и Anthropic API. Никаких "интегрируй с HubSpot/Salesforce" без флага `--enterprise`.

❌ **Маркетинговые ссылки** в коде или README — реклама себя или своего продукта = моментальный close PR.

❌ **AI-сгенерированные правки без проверки** — копипаст ChatGPT-блоков ломает методологию. Каждый PR должен быть осмыслен.

## Процесс

### Issue → Discussion → PR

**Малые правки** (typo, ссылка, опечатка) — сразу PR.

**Средние правки** (новый skill, новый business type, refactor) — сначала open issue для обсуждения. Это спасает от ситуации "я неделю работал, а вы говорите 'не нужно'".

**Большие правки** (адаптация под другой рынок, EN-версия) — open issue с тегом `RFC` (Request For Comments). Согласовываем подход, потом ты делаешь.

### Branch naming

```
feature/<short-description>
fix/<short-description>  
docs/<short-description>
showcase/<your-business-name>
```

### Commit messages

Используй [Conventional Commits](https://www.conventionalcommits.org/):

```
feat(natalia-positioning): добавлен framework для Б2Б-услуг
fix(natalia-email): исправлен порядок писем в welcome-серии
docs(README): обновлены ссылки на Telegraph
showcase(stomatologia-pokrovka): кейс 12 → 47 заявок за 6 недель
```

### PR description

Шаблон в [.github/pull_request_template.md](.github/pull_request_template.md).

Минимум:
- **Что меняется**
- **Почему**
- **Как тестировать** (если skill — какой промпт запускать)
- **Скриншот / output** (если визуальное)

## Стиль кода

### Markdown (skills, README, docs)

- Заголовки H2 / H3 — без emoji в начале
- Параграф 1-3 предложения максимум
- На "ты", не на "Вы"
- Цифры > прилагательные ("21 агент" не "много инструментов")
- Запрещённые слова: "качественный", "эффективный", "оптимизировать", "решение", "комплексный", "уникальный", "профессиональный"

Полный voice-guide: [growth-system/voice.md](growth-system/voice.md).

### SKILL.md frontmatter

```yaml
---
name: natalia-<short-name>
description: <one-sentence description что делает скилл и когда применять>
version: 1.0.0
---
```

### Тестирование skill локально

```bash
# Скопировать в ~/.claude/skills/
mkdir -p ~/.claude/skills/<skill-name>
cp skills/<skill-name>/SKILL.md ~/.claude/skills/<skill-name>/

# Перезапустить Claude Code
# Запустить skill
/<skill-name>
```

Если Claude не "видит" skill — проверь:
- frontmatter валиден (yaml-синтаксис)
- description чёткий и однозначный
- папка имеет правильное имя

## Code of conduct

Кратко: **уважение, профессионализм, никакого harassment**.

- Критикуй идеи, не людей
- Если не согласен — обоснуй цифрой / методологией / ссылкой
- "Наталья не согласна" не аргумент — для open-source каждый PR обсуждается на равных
- Английский / русский — оба ок в issues и PR
